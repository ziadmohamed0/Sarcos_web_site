Introduction

Project Essence

SarcoOS is a high-performance, modular firmware ecosystem designed to synchronize human kinetic movement with robotic precision. Developed with an emphasis on Industrial IoT (IIoT) standards, it facilitates a seamless teleoperation experience through a robust ESP32-based suit that communicates over a distributed MQTT mesh.

Real-Time Control

Low-latency teleoperation with FreeRTOS-powered ESP32 nodes ensuring deterministic response times.

IIoT Standards

Built on QoS-aware MQTT messaging with dual-broker redundancy for mission-critical reliability.

Modular HAL/MCAL

Strict hardware abstraction layers decoupling high-level logic from silicon, ensuring cross-platform portability.

ROS 2 Autonomy

Native integration with ROS 2 for advanced path planning, SLAM, and autonomous navigation via Nav2.

IIoT Dashboard

Real-time Node-RED HMI with telemetry monitoring, dual-arm control, and mission coordination.

Custom PCB Design

KiCad-designed control board with TMC2209 stepper drivers and optimized high-current power distribution.

Open Source Project

Complete source code, firmware, CAD models, PCB designs, and ROS 2 workspace available on GitHub

Sensor Node (Node A)

Acquires environmental data from MPU6050 IMU, NEO-7M GPS, and 4x HC-SR04 ultrasonic sensors. Applies DSP filters (complementary filter for IMU, median filter for GPS) and publishes via MQTT. Includes captive portal provisioning, heartbeat monitoring, and UART bridge to STM32.

IMU: MPU6050, I2C, 10 Hz GPS: NEO-7M, UART, 1 Hz Ultrasonic: 4x HC-SR04, 5 Hz

Control Node (Node B)

Drives all robotic actuators: 2x BTS7960 H-bridges for differential drive, 4x Feetech servos (95kg shoulder, 35kg elbow) for dual-arm manipulation, 2x NEMA23 + TB6600 steppers for wrist joints. PID closed-loop control for precision positioning.

Drive: 2x BTS7960, PWM Arm: 4x Servos, 50 Hz PWM Wrist: 2x NEMA23, TB6600

Journey

Project Timeline

From concept to completion — the development journey of SarcoOS

Phase 1

Concept & Research

Defining the teleoperation architecture, selecting MCU platforms, and evaluating IIoT communication protocols.

Phase 2

Firmware Development

Building the ESP-IDF modular HAL/MCAL stack with FreeRTOS task scheduling, MQTT integration, and sensor drivers.

Phase 3

Hardware Design

SolidWorks CAD modeling of the V2 robot, KiCad PCB layout with TMC2209 integration, and 3D printing of structural components.

Phase 4

Dashboard & HMI

Node-RED flow development with real-time telemetry, rover controls, arm telemetry gauges, and gripper toggles.

Phase 5

ROS 2 Integration

Building the ROS 2 workspace with MQTT bridge, Nav2 autonomous navigation, differential drive control, and URDF models.

Phase 6

Testing & Deployment

System integration testing, latency benchmarking, calibration of PID controllers, and deployment of the full teleoperation stack.
